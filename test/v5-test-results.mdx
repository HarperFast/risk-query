## Manual Tests

### 1. Insert and Replication

Inserted a record on each node via `PUT /risq/{id}`, verified the response on the originating node, then confirmed replication to all 3 remaining nodes. All 4 directions passed: central→all, lax→all, mia→all, ord→all.

Example (central-1):
```
PUT /risq/test-001  {"di": "abc123", "d": "allow", "r": 60}  → 204
GET /risq/test-001  → {"deviceId":"abc123","decision":"allow","riskScore":60,"correlationId":"test-001"}
Replicated to lax-1, mia-1, ord-1 - confirmed identical response on all.
```

### 2. GET via /RisqTable

Retrieved individual records and full record listing via the direct REST endpoint on all 4 nodes. All returned consistent data.

```
GET /RisqTable/test-001 → {"deviceId":"abc123","decision":"allow","riskScore":60,"correlationId":"test-001"}
GET /RisqTable/         → [4 records, identical across all nodes]
```

### 3. Upsert and Replication

Updated existing records on each node with new values, verified the update locally, then confirmed it replicated to all other nodes.

Example (central-1):
```
PUT /risq/test-001  {"di": "updated-device", "d": "deny", "r": 99}  → 204
GET /risq/test-001  → {"deviceId":"updated-device","decision":"deny","riskScore":99,"correlationId":"test-001"}
Replicated to lax-1, mia-1, ord-1 - all updated.
```

Repeated for lax-1, mia-1, ord-1 - all passed.

### 4. Edge Cases

**Missing fields** - PUT with only `di`, omitting `d` and `r`:
```
PUT /risq/edge-missing  {"di": "only-device"}  → 204
GET → {"deviceId":"only-device","correlationId":"edge-missing"}
```
Missing fields are omitted from the response, not null. Replicated to all nodes.

**Empty body** - PUT with `{}`:
```
PUT /risq/edge-empty  {}  → 204
GET → {"correlationId":"edge-empty"}
```
Only the correlationId (from the URL path) is stored. Replicated to all nodes.

**Nonexistent record:**
```
GET /risq/does-not-exist → null
```

**No auth:**
```
GET /risq/test-001 (no Authorization header)
→ {"type":"error:AccessViolation","code":"AccessViolation","title":"Must login","status":401}
```

### 5. Delete and Replication

Confirmed record existed on all 4 nodes, deleted from central-1, verified null on all 4.

```

GET on central-1 → null
GET on lax-1, mia-1, ord-1 → null
```
